# Finished Makes — gallery

A static photo gallery: one large centred image, a thumbnail strip
below, left/right navigation (buttons, keyboard arrows, or thumbnail
clicks).

## Folder structure

```
index.html          the page
style.css            styling
script.js             gallery behaviour
build_manifest.py     scans images/ and writes manifest.json
manifest.json         list of images, regenerated by the script above
images/               your photos go here
.github/workflows/    optional auto-deploy on push
```

## Manual update

1. Drop new photos into `images/`. Name the files descriptively —
   the title shown under the main image is generated from the
   filename, e.g. `Blue-octopus-amigurumi.jpg` becomes
   "Blue octopus amigurumi".
   To control display order, add a zero-padded number prefix, e.g.
   `01-blue-octopus-amigurumi.jpg`, `02-mustard-bear-amigurumi.jpg`,
   `10-teal-frog-amigurumi.jpg`. The number sets the order but is
   stripped from the title shown on the page. Zero-padding matters —
   without it, `10-` would sort before `2-`.
2. Run `python build_manifest.py` to regenerate `manifest.json`.
3. Upload the whole folder (or just the changed files plus
   `manifest.json`) to your webspace.

## Optional: fully automatic deploy

The included `.github/workflows/deploy.yml` rebuilds the manifest
and rsyncs the whole site to your webspace on every push to `main`.

1. Put this folder in a Git repository (GitHub).
2. In the repo's Settings → Secrets and variables → Actions, add:
   - `DEPLOY_HOST` — your webspace's SSH host
   - `DEPLOY_USER` — SSH username
   - `DEPLOY_SSH_KEY` — private key with access to that account
   - `DEPLOY_REMOTE_PATH` — target folder on the server
3. From then on: drop images into `images/`, commit, push.
   The Action does the rest.

## Notes

- Supported formats: `.jpg`, `.jpeg`, `.png`, `.webp`.
